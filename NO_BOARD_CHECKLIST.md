# 无板推进清单

## 已落地基础设施
- `code/debug_tools/mock_gnss.[ch]`
  - 提供固定轨迹的 GNSS 回放
  - 默认在 `user/cpu0_main.c` 上电后开启
- `code/debug_tools/diag_log.[ch]`
  - 周期输出 GPS / 轨迹记录 / PID / 录制参数状态
  - 便于串口看运行趋势

## 上板前建议验证

### 1. 主流程
- [ ] 进入主循环后能持续输出 `[diag]` 日志
- [ ] mock 轨迹经纬度会循环变化
- [ ] 打开轨迹记录后，`points` 数量会递增
- [ ] 停止记录后，点数保持稳定

### 2. 参数
- [ ] PID 参数读取正常
- [ ] 轨迹记录参数（距离/间隔/卫星数/最大速度）日志里能看到
- [ ] 修改参数后，日志能反映新值

### 3. 保护
- [ ] 关闭 mock 后，系统不会异常卡死
- [ ] 无有效 GNSS 时，不会误记录轨迹
- [ ] 轨迹为空时保存 Flash 有明确提示

### 4. 上板后第一轮实车
- [ ] 先关 `mock_gnss_set_enabled(1U)`，改为 `0U`
- [ ] 只验证真 GNSS 数据是否进来
- [ ] 再验证轨迹记录是否触发
- [ ] 最后再联调 WiFi telemetry / 菜单参数保存

## 下一步建议
1. 给 mock 扩成“多场景脚本”（直线/转弯/停车/丢星）
2. 给日志加级别和开关
3. 如果后面恢复车模控制，再继续抽象成：
   - 感知输入层
   - 决策状态机层
   - 执行器输出层
